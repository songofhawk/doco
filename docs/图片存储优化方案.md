# 图片存储优化方案

## 背景

**优化前的问题：**
- 图片以 Base64 编码嵌入 Yjs 文档，存储在 `ydoc_updates` 表的二进制字段中
- 单张图片会让数据库体积膨胀 1.33 倍（Base64 编码开销）
- 无法单独管理图片资源（去重、压缩、迁移）
- 难以扩展到对象存储（OSS）或 CDN

## 设计目标

1. 图片文件独立存储，数据库仅记录元数据
2. 通过 URL 引用图片，便于后续迁移到云存储
3. 保持编辑器使用体验不变
4. 为未来扩展预留接口（图床、OSS、CDN）

## 技术方案

### 1. 数据模型

新增 `Attachment` 表：

```python
class Attachment(Base):
    __tablename__ = "attachments"
    id = Column(String, primary_key=True)      # UUID
    filename = Column(String, nullable=False)   # 原始文件名
    filepath = Column(String, nullable=False)   # 本地存储路径
    mime_type = Column(String, nullable=False)  # MIME 类型
    size = Column(Integer, nullable=False)      # 文件大小（字节）
    doc_id = Column(String, ForeignKey("documents.id"), nullable=True)  # 关联文档
    created_at = Column(DateTime, default=datetime.utcnow)
```

### 2. 后端 API

#### 上传接口
```
POST /api/attachments/upload
Content-Type: multipart/form-data

参数：
- file: 图片文件
- doc_id: 文档 ID（可选）

返回：
{
  "id": "uuid",
  "url": "/api/attachments/{id}"
}
```

**实现要点：**
- 文件保存到 `backend/uploads/{uuid}{ext}`
- 使用 UUID 避免文件名冲突
- 记录元数据到数据库

#### 访问接口
```
GET /api/attachments/{id}
```

**实现要点：**
- 返回 `FileResponse`，设置正确的 `Content-Type`
- 支持浏览器缓存（后续可加 ETag）

### 3. 前端改动

修改 `/` 命令中的图片插入逻辑：

**优化前：**
```typescript
const reader = new FileReader()
reader.onload = () => {
  editor.chain().focus().setImage({
    src: reader.result as string  // Base64
  }).run()
}
reader.readAsDataURL(file)
```

**优化后：**
```typescript
const formData = new FormData()
formData.append('file', file)
const res = await fetch('http://127.0.0.1:8000/api/attachments/upload', {
  method: 'POST',
  body: formData
})
const data = await res.json()
editor.chain().focus().setImage({
  src: `http://127.0.0.1:8000${data.url}`  // URL 引用
}).run()
```

## 存储对比

| 方案 | 存储位置 | 单张 1MB 图片占用 | 优势 | 劣势 |
|------|---------|------------------|------|------|
| Base64 | SQLite 二进制字段 | ~1.33MB | 无需额外服务 | 数据库膨胀、难以管理 |
| 文件存储 | 本地文件系统 | 1MB | 独立管理、易扩展 | 需要文件系统访问 |

## 扩展路径

### 阶段 1：本地文件存储（当前）
```
前端 → FastAPI → 本地 uploads/ 目录
```

### 阶段 2：对象存储（未来）
```python
# 修改上传接口，支持多种存储后端
class StorageBackend:
    def upload(self, file) -> str:
        pass

class LocalStorage(StorageBackend):
    def upload(self, file):
        # 保存到本地
        return f"/api/attachments/{id}"

class OSSStorage(StorageBackend):
    def upload(self, file):
        # 上传到阿里云 OSS
        return f"https://cdn.example.com/{id}"
```

## 部署注意事项

1. **数据库迁移**：重启后端自动创建 `attachments` 表
2. **文件目录**：确保 `backend/uploads/` 目录可写
3. **CORS 配置**：已配置允许跨域上传
4. **备份策略**：需同时备份数据库和 `uploads/` 目录
