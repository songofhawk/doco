# 我用 AI 手搓了一个编辑器，终于不用忍受别人的规则了

事情要从去年说起。

那段时间我写了很多文档——技术方案、读书笔记、项目复盘、需求文档。切换了无数个工具，每次都卡在同一个地方：写的时候挺爽，想拿走就难了。

我不是没试过各种方案。只是试得越多，越觉得不对劲。

## 那些"够用"的工具，其实都在圈地

**语雀**是我用过最舒服的编辑器。排版干净、交互细腻，中文编辑体验几乎无可挑剔。我在上面写了上百篇技术文档，每一篇都很开心——直到我需要把它们批量导出给 AI 做二次处理。点开导出菜单，傻眼了。单篇导出还行，批量就是做梦。更不用说我想写个脚本自动备份、想让 Agent 帮我整理笔记、想把文档接入自己的发布流程……全都不行。不是技术上做不到，是这条路径压根就没设计给你走。

数据是你的吗？是。但你只能在平台划定的边界内消费它。越过那条线，什么都没有。

然后是**飞书文档**。这可能是全行业做得最重的文档产品——不是贬义，它是真的好用。实时协同丝滑流畅，多维表格和电子表格无缝嵌入，块引用、目录、评论系统一应俱全。但飞书的问题和它的优点一样大：它是一整个生态。你要用飞书文档，就得接受它的 IM、日历、审批、OKR——它像一家五星级酒店，住进去什么都有，但你改不了墙纸，调不了空调，Wi-Fi 密码都是前台说了算。我想做的只是一个编辑器，不是移民到一个新国度。

Notion 太重，Google Docs 太素，Obsidian 编辑体验差了口气。每个工具都在自己的赛道上做到极致，但全都绕开了我最核心的需求：**一个真正把数据所有权还给用户的在线文档**。

我想要的东西其实就两条：

1. 像语雀一样流畅的富文本写作体验。
2. 数据完全归我——怎么存、怎么导出、怎么被脚本调用、怎么给 Agent 喂，全部我说了算。

听起来简单到可笑对吧？就这两条，找遍了，没有。

那就自己造吧。

## Doco 是什么

Doco，取"文档"的谐音。没做产品定位，没画用户画像，没有 PRD——就是我自己需要一个工具，然后我把它做了出来。

但写着写着，它已经不只是个"够用"的编辑器了。

### 编辑这件事，流畅是底线

内核选了 Tiptap（ProseMirror 的 React 封装），开源世界最成熟的富文本框架。但骨架谁都能搭，拉开差距的是肌肉记忆层面的细节：

- 输入 `/` 弹出命令面板，打拼音缩写就能搜——`dmk` 插入代码块，`lct` 插入流程图，`glk` 插入高亮提示。手不离开键盘，一气呵成。
- 选中文字，浮动工具栏自动弹出。加粗、斜体、链接、替换为标题——该有的都在指尖两厘米以内。
- 鼠标悬停在任意段落左侧，拖拽手柄浮现。像 Notion 一样拖拽调整顺序，松手即就位。
- 写长文档的时候，把暂时不看的章节折叠收起来。折叠状态会记住——下次打开，还是你上次看到的那个样子。
- 标题自动编号。H1、H2、H3 自动按层级标记为 1、1.1、1.1.1。按一下快捷键切换，再也不用手动维护编号。

### 画图这件事，代码就够了

写技术文档最怕什么？流程图画好了，要改了。

传统做法：打开 draw.io，导出 PNG，插入文档。要改？回到 draw.io，改完重新导出，重新插入，重新对齐。每次更新都像做一场小手术。

Doco 的做法：在文档里插入一个 Mermaid 或 PlantUML 代码块，把源码写进去，图表自动渲染出来。流程图、时序图、类图、甘特图、状态图——只要是 Mermaid 能画的，全支持。双击随时编辑源码，渲染结果实时更新。放大查看、拖拽平移，跟独立图表工具一样的体验。

你要的图和你写的话，放在同一个文档里。源码就是图，图就是源码。

### 电子表格，不小心写深了

一开始我只是想给文档加个简单表格。

后来想，表格里能不能算个求和？加了。能不能冻结表头？加了。能不能筛选排序？加了。等着回过神来，我已经实现了一个完整的电子表格引擎——公式计算、单元格格式化、合并拆分、CSV 导入导出、协同编辑，全有了。

它可以嵌在文档中以块的形式存在，也可以作为独立的全屏电子表格打开。

说实话，这不是我计划好的。只是在写技术方案的时候，发现经常需要随手算一笔数；在整理对比调研的时候，需要一个结构化的矩阵。传统工具里这种事只能打开 Excel，另起一个文件，做好之后截图贴回来。太割裂了。

现在它就在我的编辑器里，跟正文活在同一页上。

### 多端同步，但服务器是我自己的

协同编辑这一层用了 Yjs 的 CRDT 算法。说人话就是：

- 电脑上写一半，关屏出门，掏出手机接着写。没有保存按钮，没有同步冲突，一切自动发生。
- 你和你同事同时编辑同一篇文档，不存在"冲突版本"这个东西——所有修改自动合并且保证最终一致。
- 断网也能正常编辑。浏览器的 IndexedDB 是主存储，后端只是快照备份。网络恢复后，差异自动同步。

但更关键的是：**后端在我自己手里。**

没有 SaaS 平台，没有供应商绑定。数据跑在我自己的服务器上，存成 SQLite 文件，想备份就备份，想迁移就迁移，想分析就分析。WebSocket 也是我自己的。不存在"服务倒闭数据取不出"这种荒谬的事。

### 导出不是功能，是权利

我一直觉得，一个文档工具的导出能力是这个工具是否尊重用户的试金石。

Doco 里的导出不是你跪着求平台赏给你的恩赐：

- **单篇 Markdown 导出**：走 API 调用，拿到干净的 `.md` 原文。Mermaid 和 PlantUML 图表自动转为围栏代码块，表格保留结构，电子表格降级为 CSV 块。
- **全知识库 ZIP 打包**：一键下载，文件夹层级完整保留，每个文档一个 `.md` 文件。整库迁移，没有阻力。
- **前端即时导出**：编辑器里直接导出 Word、PDF，所见即所得，不用切到导出页面。

导出的 Markdown 拖进 Git 仓库就能版本管理，喂给 AI 就能二次加工，丢进静态网站生成器就能变成博客。你的文档终于不再是谁的俘虏——它是你真正的数字资产。

### API 优先：你的文档是一堆可编程的数据

Doco 在 Web 界面之外，还暴露出了一套完整的 REST API，OpenAPI 3.1 规范，Bearer Token 认证，支持版本控制。

这意味着你可以做这些事情：

```
# 写个 cron job，每天自动备份所有文档到 GitHub
# 搭个 Agent，让它读取你的知识库，帮你归类、摘要、关联
# 从你的发布流程里直接调 API，把文档同步到博客
# 用你喜欢的任何语言读取、修改、创建文档
```

文档放在文件夹里只是死字节。通过 API，它变成了你可以编程处理的信息资产。这才是真正的"数据属于我"——不只是能下载，是能用代码驱动。

## 说真的，感谢 AI

要是放在两年前，我一个人绝对不敢做这种项目。

你看这都包含了什么：React 前端、Node.js 后端、ProseMirror 编辑器框架、Yjs CRDT 协同算法、SQLite 数据库、Mermaid 和 PlantUML 的图表渲染、一个完整的电子表格引擎、WebSocket 实时通信、OAuth 和邮箱登录、API Token 鉴权、Markdown/Word/PDF 的导入导出……这每一个模块放到传统开发里，都至少需要专门研究两天才能上手。

但现在有 AI。

ProseMirror 的 Node 和 Mark 搞不清楚？AI 帮你把文档读透了。Yjs 的 UndoManager 跟协同模式冲突？AI 帮你排查。电子表格公式解析器怎么写？AI 跟你一起推演。很多时候我的角色已经不是"写代码的人"了——我更像一个架构师，负责判断方向、做决策、拍板，然后 AI 去把代码落下来。

以前大家开玩笑说"我有个好想法，就差一个程序员了"——那是自嘲。现在不是了。

**Doco 就是对这句话最好的回答。**
